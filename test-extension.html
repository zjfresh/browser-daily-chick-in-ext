📘 Daily Reminder Extension - 测试页面

🎯 测试目的

此页面用于验证 Daily Reminder Extension 是否正常工作。请按照以下步骤进行测试。

📋 安装检查清单

  • 打开 chrome://extensions/
  • 开启"开发者模式"
  • 点击"加载已解压的扩展程序"
  • 选择项目文件夹 browser-daily-chick-in-ext
  • 确认扩展出现在列表中且已启用
  • 检查浏览器工具栏是否出现扩展图标

✅ 功能测试步骤

1. 基础功能测试

操作：点击扩展图标打开弹出窗口

期望：看到统计信息和操作按钮

2. 配置管理测试

操作：点击"⚙️ 管理配置"打开选项页面

期望：能够添加、编辑、删除配置

3. 导入测试配置

操作：在选项页面点击"导入配置"，选择 demo-config.json

期望：成功导入测试配置

4. Toast 提醒测试

操作：在弹出窗口点击"🔄 立即检查"

期望：如果有配置需要触发，会显示confirm对话框

5. 自动触发测试

操作：打开新标签页或刷新页面

期望：如果有配置需要触发，会自动显示提醒或打开新页面

6. 🌅 跨天触发测试

功能：当页面保持打开状态过夜时，新的一天开始后会自动检查配置

测试方法：

  • 保持此页面打开
  • 切换到其他标签页或最小化浏览器
  • 等待跨越午夜（或修改系统时间）
  • 重新聚焦到此页面
  • 检查控制台日志，应该看到日期检查和配置触发信息

期望：新的一天开始时，满足条件的配置会自动触发

7. 🕐 定时检查测试

功能：后台脚本每小时检查一次日期变化

测试方法：

  • 打开 chrome://extensions/
  • 找到扩展，点击"Service Worker"
  • 在控制台中查看定时检查日志

期望：每小时看到 "[Background] Daily check alarm triggered" 日志

⚠️ 注意事项

  • 某些页面（如 chrome:// 页面）不会显示提醒
  • 第一次触发后，同一天不会重复触发（除非重置）
  • 可以在弹出窗口点击"重置今日"来重新触发测试
  • 测试时建议使用"每天"触发规则以便快速验证
  • 新功能：页面焦点变化和可见性变化都会触发日期检查

🐛 问题排查

如果扩展无法加载：

  • 检查 manifest.json 格式是否正确
  • 确认所有文件都在同一目录下
  • 查看扩展页面的错误信息

如果提醒不显示：

  • 确认当前页面不是 chrome:// 页面
  • 检查配置的触发规则是否满足
  • 查看浏览器开发者工具的控制台错误
  • 检查后台脚本日志：打开扩展页面 → Service Worker

如果跨天触发不工作：

  • 确认后台脚本正在运行（Service Worker 活跃）
  • 检查浏览器权限设置
  • 查看后台脚本和内容脚本的控制台日志
  • 尝试手动切换标签页或重新聚焦窗口

如果弹出窗口空白：

  • 右键弹出窗口选择"检查"查看错误
  • 确认 utils.js 正确加载

🎉 测试成功标志

如果以下功能都正常工作，说明扩展安装成功：

  • 弹出窗口正常显示统计信息
  • 能够在选项页面添加/编辑配置
  • Confirm 提醒能够正常显示和交互
  • 自动打开功能正常工作
  • 导入/导出配置功能正常
  • 新功能：跨天自动触发配置检查
  • 新功能：页面焦点/可见性变化触发检查
